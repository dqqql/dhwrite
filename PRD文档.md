# 匕首之心团前共创工具 PRD

更新时间：2026-04-29

## 1. 产品概述

这是一个围绕《匕首之心》团前准备场景设计的多人共创白板工具。

核心体验：

- 玩家进入同一房间
- 从卡组中抽取地点 / 人物 / 特色卡
- 将卡牌拖放到共享地图
- 通过地点疆域、连线关系、文字标注共同构建世界、派系与冲突

目标是帮助 GM 和玩家在正式开团前，用一轮或多轮结构化共创快速搭出可用的战役起点。

## 2. 房间生命周期

### 三态模式

系统采用三态流转：

1. `free`
   - 玩家进入房间后的默认模式
   - 可自由查看、整理、标注、连线、编辑地图卡
2. `co-creation`
   - 由房主点击“开始共创”进入
   - 发手牌、开启回合、允许抽牌与打牌
3. `normal`
   - 由房主点击“结束共创”进入
   - 回收未打出的手牌，保留地图成果
   - 允许继续整理地图并进行导出

### 流程

1. 房主创建房间，默认全选官方卡包，可稍后调整
2. 玩家通过邀请码加入房间
3. 所有人先在 `free` 模式中浏览、整理或预备
4. 房主开始共创，进入 `co-creation`
5. 玩家轮流抽牌、打牌、创建自定义卡、编辑地图
6. 房主结束共创，进入 `normal`
7. 房主可再次开始新一轮共创

### 备份提醒

房间仅临时保留 3 天，因此需要在以下时机提醒用户备份：

- 房主创建房间时
- 玩家首次进入房间时
- 房主结束共创时

## 3. 地图与交互

### 画布

- 无限画布
- 支持平移与缩放
- 默认背景为浅色点阵 / 网格白板风格

### 图层

- 背景层：地点疆域
- 实体层：地图卡片
- 连线 / 标注层：关系线、文字标注
- 浮层：上下文菜单、关系标签气泡、提示面板

### 卡片操作

- 手牌可直接拖放到地图
- 卡片可移动、展开、缩放
- 卡片支持右键上下文菜单
- 自由模式下可物理删除地图卡
- 共创模式下地图卡以“回收”替代“删除”
- 地点卡打出后自动生成疆域范围
- 地点疆域支持独立拖拽与缩放

### 卡片颜色编辑

- 卡片在编辑时可修改展示色
- 默认颜色倾向由类型决定，但最终显示色以卡片实例自身颜色为准
- 修改后的颜色应同步作用到：
  - 地图卡片
  - 地点疆域
  - 手牌
  - 抽牌预览
  - 打牌 / 回收相关动画

### 双态显示

- 微缩态：显示类型图标、类型标签、标题、所属玩家颜色
- 展开态：显示完整标题与正文描述

## 4. 连线与标注

### 连线系统

- 玩家可在任意两张卡牌之间建立有向连线
- 连线颜色用于表示关系类型：
  - 红色：冲突 / 敌对
  - 绿色：盟友 / 友善
  - 灰色：未知 / 待定
- 连线支持关系标签，如“效忠”“敌对”“血脉”“传闻”

### 关系标签展示要求

- 标签默认应显示在线条曲线的中段附近
- 感知上优先保持“比较居中”，不允许明显贴近任一端点
- 当两张卡距离过近时，标签应自动外移以保持可见
- 标签不应被卡片主体遮挡
- 标签本身仍应可点击并进入关系编辑

### 标注系统

- 玩家可在地图任意位置创建文字标注
- 标注独立于卡牌存在
- 支持字号调整

## 5. 卡牌系统

### 类型

卡牌分三类：

- 地点 `Location`
- 人物 `NPC`
- 特色 `Feature`

面向用户的显示文案统一使用中文：

- 地点
- 人物
- 特色

### 默认颜色倾向

- 地点：绿色
- 人物：蓝色
- 特色：紫色

### 手牌与抽牌

- 开始共创时，每位在线玩家获得 5 张初始手牌
- 每回合可执行一次“三选一”抽牌
- 玩家每回合可打出任意数量手牌
- 玩家可创建自定义卡并加入自己的手牌

### 发牌去重

发牌不得与以下卡重复：

- 任何玩家当前手牌中的卡
- 地图上已存在的卡

### 共创结束回收

房主结束共创后：

- 未打出的官方 / 导入卡回到牌堆
- 未打出的自定义卡直接删除
- 已在地图上的卡片全部保留

## 6. 回合与协作规则

### 当前回合玩家可做

- 打牌
- 抽牌
- 创建自定义卡
- 编辑地图卡
- 新建连线
- 结束回合

### 非当前回合玩家可做

- 编辑已有地图卡
- 调整位置 / 缩放 / 地点疆域
- 创建连线
- 添加标注
- 创建自定义卡到自己的手牌

限制：

- 非当前回合玩家不能抽牌
- 非当前回合玩家不能打牌

### 在线状态

- 玩家列表应始终可见
- 在线玩家高亮，离线玩家置灰
- 回合轮到离线玩家时自动跳过

### 房主转移

- 房主离线时自动转给下一位在线玩家
- 新房主立即拥有完整房主权限

### 软锁

- 某玩家正在编辑或拖拽卡片时，其他玩家不能同时修改该卡
- 软锁应在操作结束或超时后释放

## 7. 房间系统

- 一个站点可存在多个独立房间
- 每个房间通过邀请码进入
- 创建房间时默认全选官方卡包
- 房间状态自动保存
- 房间默认保留 3 天，到期自动删除

## 8. 卡包系统

### 官方卡包

- 官方内容以多个内置卡包提供
- 房间可选择启用哪些官方卡包

### 导入

支持两种导入：

- 整包导入 `.dhpack.json`
- 从卡包库中按卡选择导入

### `.dhpack.json` 结构

```json
{
  "format": "dhpack",
  "version": 1,
  "pack_name": "卡包名称",
  "cards": [
    {
      "type": "Location",
      "title": "灰望烽台",
      "content": "烧焦山脊上的旧烽火塔仍会在无风夜里自行点亮。",
      "style": "#22c55e"
    }
  ]
}
```

## 9. 房间备份

### `.dhroom.json`

房间备份应包含：

- 房间元信息
- 当前模式
- 当前房主
- 当前回合玩家
- 回合顺序
- 玩家列表
- 每位玩家的手牌
- 地图卡片
- 连线
- 标注
- 选中卡包
- 房间设置

### 备份恢复要求

- 必须支持在共创中途恢复
- 必须保留地图、连线、标注与玩家手牌
- 牌堆顺序允许重建，但可用牌集合必须正确

## 10. 导出能力

支持三类导出：

1. 地图卡片导出为 `.dhpack.json`
2. 房间完整备份导出为 `.dhroom.json`
3. 导出叙事摘要 Markdown，便于 GM 复制进笔记工具

## 11. MVP 范围

MVP 应覆盖：

- 三态房间流转
- 多人房间 + 邀请码加入
- 玩家在线 / 离线展示
- 房主自动转移
- 地图平移缩放
- 卡片打出、移动、缩放、展开
- 地点疆域独立编辑
- 抽牌三选一
- 自定义卡
- 软锁
- 连线与关系标签
- 文字标注
- 卡包导入
- 房间备份导出
- 叙事摘要导出

## 12. 近期同步到 PRD 的实现约束

这轮需要明确写进需求的行为有两条：

1. 卡片颜色编辑不是局部效果，而是整张卡在所有展示面的统一视觉来源。
2. 连线关系标签必须优先保持居中感知，并在近距离场景下依然可见、可点、不会被卡片遮挡。
