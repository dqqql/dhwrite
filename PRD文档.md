# 匕首之心 · 团前共创工具 PRD

---

## 1. 产品概述

一个基于卡牌驱动的多人共创白板，帮助 GM 和玩家在《匕首之心》团前快速构建疆域、人物关系和核心冲突。

**核心体验：** 类似《杀戮尖塔》的手牌系统 + Figma 式多人白板。玩家从预制卡牌库中抽取随机卡牌，轮流打出到共享地图上，通过拖动范围、连接线和文字标注，逐步构建战役舞台。

---

## 2. 核心流程

### 2.1 房间生命周期

系统只有两种模式：**自由模式** 和 **共创模式**。房主通过"开始共创"和"结束共创"在两个模式间切换，支持多轮共创。

1. **创建房间** — 房主选择卡包（默认全选，进入后可调整），生成邀请码。创建完成时弹出一次备份提醒，告知房间仅临时保存 3 天，记得在共创结束后导出。
2. **自由模式** — 玩家进入后默认处于自由模式。可在地图上自由标注、连线、编辑已有卡牌，也可导出卡包或完整备份。不涉及手牌、抽牌、打牌。
3. **开始共创** — 房主点击"开始共创"，系统洗牌并发给每位玩家初始手牌（5 张），进入共创模式。
4. **共创循环** — 玩家轮流抽牌、打牌、自创卡牌，共同构建地图。
5. **结束共创** — 房主点击"结束共创"时先弹窗提示"未打出的自定义卡将删除"。确认后，系统将未打出的官方/导入卡回收至牌组，将未打出的自定义卡直接删除，回到自由模式。地图上已打出的卡牌保留。
6. **多轮共创** — 房主可再次点击"开始共创"发起新一轮，系统根据当前可用牌重新洗牌，重复步骤 3→4→5。每轮共创的产出叠加在地图上。

### 2.2 模式对比

| 特性 | 自由模式 | 共创模式 |
|------|---------|---------|
| 文字标注 | ✓ | ✓ |
| 连线 | ✓ | ✓ |
| 编辑已有卡牌 | ✓ | ✓ |
| 完整导出（备份） | ✓ | ✓ |
| 手牌 / 抽牌 | ✗ | ✓ |
| 打牌 | ✗ | ✓ |
| 自创卡牌 | ✗ | ✓ |
| 地点范围拖动 | ✗ | ✓ |
| 地图卡移除方式 | 物理删除 | 回收至手牌 |
| 导出地图卡牌为卡包 | ✓（共创结束后） | ✗ |
| 触发方式 | 进入房间默认 / 房主结束共创 | 房主点击"开始共创" |

## 3. 地图与交互

### 3.1 画布

- 无限画布，支持缩放和平移。
- 默认背景为浅色半虚线网格（类似 Figma 的 dot grid）。
- 画布状态自动保存，玩家重连后恢复。

### 3.2 图层管理

| 图层 | 内容 | 说明 |
|------|------|------|
| 背景层 | 地点卡的势力范围色块 | 半透明，可叠放，支持嵌套（如"城中有堡"） |
| 实体层 | 人物、特色等卡牌 | 浮动卡片，可拖拽、缩放 |
| 标注层 | 文字标注、连线 | 置于最上层 |

### 3.3 卡牌操作

- **打出：** 从手牌拖拽到地图，即拖出手牌区。
- **右键菜单：** 移动、缩放（方便展示过长的文本）、编辑内容、置顶/置底、锁定（防止误操作）。
- **自由模式移除：** 地图上的卡牌可被直接物理删除。
- **共创模式回收：** 地图上的卡牌不提供"删除"，改为"回收"按钮。回收后卡牌返回其原打出玩家手牌；若原打出玩家离线，仅房主可执行回收，回收后仍回到原玩家手牌。
- **地点卡特殊能力：** 打出后可通过拖拽边缘自由拉伸范围，表示为半透明色块，代表疆域或势力范围。

### 3.4 双态展示

- **微缩态（默认）：** 显示标题、类型图标、所属玩家颜色。适合画布卡牌较多时快速浏览。
- **详情态（点击展开）：** 显示完整标题和详细描述文字，支持编辑。

### 3.5 连线系统

- 玩家可在任意两张卡牌之间拉出带箭头的连线。
- 连线颜色区分关系类型：
  - 红色 — 冲突 / 敌对
  - 绿色 — 盟友 / 友善
  - 灰色 — 未知 / 待定
- 右键连线可添加"关系标签"：效忠、敌对、传闻、血脉等。

### 3.6 文字标注

- 玩家可在地图任意位置添加文字标注，独立于卡牌存在。支持修改字体大小、缩放，类似于word的文本框

## 4. 卡牌系统

### 4.1 卡牌类型与颜色

| 类型 | 默认颜色倾向 | 说明 |
|------|-------------|------|
| 地点 (Location) | 绿 | 城市、地貌、建筑、疆域等 |
| 人物 (NPC) | 蓝 | 关键角色、敌人、怪物、阵营势力等 |
| 特色 (Feature) | 紫 | 故事引子、魔法、科技、文化特征、事件等 |

卡牌暂时只显示文字（标题 + 详细描述），不做数值结算。

### 4.2 手牌与抽牌

- 每位玩家拥有独立手牌。共创模式开始时，系统自动发给每位玩家 5 张初始手牌。
- 每回合限抽 1 次：系统从牌组随机推选 3 张，玩家 3 选 1 加入手牌。
- 玩家可打出任意数量的手牌（不限于 1 张）。
- 支持玩家点击"自创"，填写空白模版创建一张新卡加入手牌，次数不限。
- 自创卡编辑界面应简洁美观，字段结构与官方卡一致，便于后续导出与复用。
- 卡牌池为房间共享牌组（所有卡包合并后洗牌）。

### 4.3 发牌去重规则

- 发出的卡牌默认不与场上已有的卡牌重合。**场上已有**包括：
  - 所有玩家当前手牌中的卡牌
  - 地图上已打出的卡牌
- 抽牌和初始发牌时，若牌组中剩余卡牌不足，则提示"牌组已空"。

### 4.4 手牌回收

- 房主点击"结束共创"后，系统回收所有玩家手牌中未打出的官方卡和导入卡，归还牌组。
- 房主点击"结束共创"时，需先弹窗提醒：**未打出的自定义卡将删除**。
- 所有玩家手牌中未打出的自定义卡在结束共创后直接删除，不进入牌组。
- 地图上已打出的卡牌保留不动。
- 回收后回到自由模式，玩家不再持有手牌。

### 4.5 卡包系统

- 官方内置 5 套战役框架（5 个卡包）。
- 创建房间时可多选卡包。
- 卡包内部按类型分类组织。
- 支持用户通过标准 JSON 格式上传自定义卡包。
- 导入支持两种方式：
  - 导入整个 `.dhpack.json` 卡包
  - 在查阅某个卡包时，导入单张卡牌 JSON，并并入当前卡包或房间可用牌池

**JSON 结构：**

```json
{
  "pack_name": "战役包名称",
  "cards": [
    {
      "type": "Location | NPC | Feature",
      "title": "标题",
      "content": "描述文本",
      "style": "#HEX颜色"
    }
  ]
}
```

### 4.6 解耦设计

卡包数据与交互逻辑分离。开发分两阶段：

1. **第一阶段：** 使用测试卡牌验证交互——抽牌逻辑、回合结算、多人同步、拖拽动画、JSON 导入导出。
2. **第二阶段：** 交互稳定后，通过 JSON 接口批量导入正式卡包内容。

## 5. 回合制规则

### 5.1 回合行动

玩家轮到自己回合时，可进行以下操作（不限顺序）：

1. **打牌** — 打出任意张手牌到地图。
2. **抽牌** — 3 选 1（每回合限 1 次）。
3. **自创** — 创建一张自定义卡牌加入手牌。
4. **编辑与连线** — 调整已有卡牌的位置、内容，或添加新的连线。
5. **跳过 / 结束回合** — 进入下一位玩家。

非当前回合玩家处于**弱回合限制**下：

- 不可执行**抽牌**与**打牌**
- 可以执行其他协作操作，包括移动卡牌、编辑内容、添加连线、添加文字标注，以及自创卡牌加入自己的手牌

### 5.2 玩家悬浮窗与回合顺序

- 悬浮窗始终显示，进入房间后即展示所有加入过的玩家（包括房主，名字前加粗或特殊符号以区别）。
- **在线状态：** 在线玩家名字标亮（绿色或高亮），离线玩家名字标暗（灰色）。
- **回合跳过：** 回合轮到离线玩家时自动跳过，顺延至下一位在线玩家。
- 按照悬浮窗列表顺序行动，默认从上至下，房主可以拖拽调整顺序。
- **强制跳过：** 当前回合玩家长时间无响应时，房主可执行强制跳过，直接轮到下一位玩家。
- 玩家断开连接后仍保留在列表中（标暗），重连后恢复标亮。

### 5.3 房主转移

- 房主掉线时，房主权限自动转移给玩家列表中的下一位在线玩家。
- 若当时没有其他在线玩家，则房主权限保留给原房主，待其重连后恢复。
- 自动转移后的房主拥有完整房主权限，包括开始/结束共创、强制跳过、调整回合顺序、移除玩家、删除房间。

### 5.4 协作防护（软锁定）

- 当某位玩家正在拖拽或编辑某张卡牌时，该卡牌在其他玩家端显示为"不可操作"状态。
- 编辑完成后（或超时自动释放），锁定解除。

## 6. 房间系统

- 同一网站下可创建多个房间，互不干扰。
- 房间通过唯一邀请码（或 URL）进入。
- 创建房间时默认全选卡包，进入后可以通过房间设置去除。
- 房间状态持久化：自动保存。玩家断开重连后恢复手牌和地图状态。
- 房间为**临时存储**，自创建时起固定保留 3 天，到期后自动删除，以控制存储成本。
- 备份提醒弹窗触发 3 次：房主创建房间时、玩家首次进入房间时、房主结束共创时。
- 房主权限：移除玩家、调整回合顺序、强制跳过、删除房间。

## 7. 导出功能

系统区分两种导出格式：**卡包格式** 和 **房间备份格式**，互不混淆。

### 7.1 卡包格式（.dhpack.json）

仅包含卡牌内容数据，不含房间状态。用于复用卡牌、分享卡包、导入到其他房间。

```json
{
  "format": "dhpack",
  "version": 1,
  "pack_name": "导出卡包名称",
  "cards": [
    {
      "id": "唯一标识",
      "type": "Location | NPC | Feature",
      "title": "标题",
      "content": "描述文本",
      "style": "#HEX颜色"
    }
  ]
}
```

### 7.2 房间备份格式（.dhroom.json）

包含房间完整状态——模式、当前回合、玩家手牌、卡牌、位置、连线、标注、玩家信息，用于完整备份或迁移房间。

备份恢复要求：

- 必须支持在共创进行到一半时恢复房间
- 必须保留所有玩家当前未打出的手牌
- 不要求保留牌库剩余顺序；恢复后可根据剩余可用卡重建并重新洗牌

```json
{
  "format": "dhroom",
  "version": 1,
  "room": {
    "name": "房间名",
    "invite_code": "邀请码",
    "created_at": "ISO时间戳",
    "expires_at": "ISO时间戳"
  },
  "session": {
    "mode": "free | co-creation",
    "current_host": "玩家昵称",
    "current_turn_player": "玩家昵称 | null",
    "turn_order": ["玩家A", "玩家B"],
    "hands": [
      {
        "owner": "玩家昵称",
        "cards": [
          {
            "id": "唯一标识",
            "type": "Location | NPC | Feature",
            "title": "标题",
            "content": "描述文本",
            "style": "#HEX颜色",
            "is_custom": false
          }
        ]
      }
    ]
  },
  "map": {
    "cards": [
      {
        "id": "唯一标识",
        "type": "Location | NPC | Feature",
        "title": "标题",
        "content": "描述文本",
        "style": "#HEX颜色",
        "is_custom": false,
        "x": 0.0,
        "y": 0.0,
        "width": 200,
        "height": 150,
        "territory": { "x": 0, "y": 0, "width": 0, "height": 0 },
        "placed_by": "玩家昵称",
        "player_color": "#HEX"
      }
    ],
    "connections": [
      {
        "from_card_id": "id",
        "to_card_id": "id",
        "color": "red | green | gray",
        "label": "关系标签（可选）"
      }
    ],
    "annotations": [
      {
        "id": "唯一标识",
        "text": "标注内容",
        "x": 0.0,
        "y": 0.0,
        "font_size": 14
      }
    ]
  },
  "players": [
    {
      "nickname": "玩家昵称",
      "color": "#HEX",
      "is_host": true
    }
  ],
  "exported_at": "ISO时间戳"
}
```

### 7.3 导出时机

| 导出类型 | 格式 | 可用模式 | 说明 |
|----------|------|---------|------|
| 地图卡牌 → 卡包 | `.dhpack.json` | 自由模式（共创结束后） | 将地图上已打出的卡牌导出为新卡包，包含官方卡、导入卡与已落图的自定义卡 |
| 完整房间备份 | `.dhroom.json` | 任意模式 | 包含模式、当前回合、玩家手牌、全部卡牌、位置、连线、标注、玩家信息 |
| 叙事摘要 | Markdown | 任意模式 | 地图上所有实体+连线转化为结构化文本，方便 GM 粘贴到笔记软件 |

## 8. 玩家身份

- 进入房间时填写显示昵称。
- 系统自动分配玩家颜色（用于卡牌边框高亮和连线标识）。
- 无需注册/登录，通过邀请码 + 昵称即可加入。

## 9. 技术考量（待定）

以下技术选择在开发启动前需确定：

- **前端框架：** 需支持无限画布渲染（Canvas API / SVG / WebGL），推荐基于成熟画布库评估。
- **实时同步：** WebSocket 或 WebRTC，需支持房间级广播和状态同步。
- **后端：** 需处理房间管理、状态持久化、自动保存。
- **数据存储：** 房间状态 + 卡包数据 + 玩家手牌。
